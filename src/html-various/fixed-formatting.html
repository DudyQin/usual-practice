在"10.3.7 Absolutely positioned, non-replaced elements"中有这一句话，The constraint that determines the used values for these elements is:

'left' + 'margin-left' + 'border-left-width' + 'padding-left' + 'width' + 'padding-right' + 'border-right-width' + 'margin-right' + 'right' = width of containing block

简化版本就是position:fixed的元素的 border-box的width + margin-edge的宽度 + right值 + left值 = 包含块的width

下面这两个实例中，containing block viewport的大小是确定的，充满整个视口，也就是它的宽度是所谓的100%（CW），按照上面那个公式，现在已知fixed元素的margin-edge的宽度为0(div默认margin为0)（MW），left和right值为100（LW,RW），就差border box的宽度不知道了，而border box的宽度 = CW - MW - LW - RW = 100% - 0 - 100 - 100，而求得的这个值的结果就是包含块宽度减去200像素时的，接下来在"10.3.8 Absolutely positioned, replaced elements"中还有一句话：
If at this point the values are over-constrained, ignore the value for either 'left' (in case the 'direction' property of the containing block is 'rtl') or 'right' (in case 'direction' is 'ltr') and solve for that value.
也就是说，在direction为ltr的情况下(这里就是)，right值将被忽略，right值将存在，但是不是它显式的起作用引起的,因此元素包括margin-edge部分会往右边移100px，而right实际是参与border-box的width的计算了的，因此右边会空余一部分 如下所示：
